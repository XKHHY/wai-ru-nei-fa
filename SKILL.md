# 外儒内法 — 个人自适应行为框架

> 基于 星空的后花园 的方法论。外儒（Confucian exterior）降低摩擦，内法（Legalist interior）守卫边界。

## When to use this skill

Load when the user mentions:
- Workplace decisions, interpersonal judgment, identity positioning, expectation management
- 职场决策, 人际判断, 身份定位, 期望管理
- 外儒内法, 控距 (distance control), 迎合 (alignment), 策略层/表达层 (strategy/expression layer)
- 日结工/学徒/合伙人 (day laborer / apprentice / partner identity types)
- 情绪标签 (emotion labeling), 功耗分配 (energy allocation)
- 课题分离 (task separation), 边界管理 (boundary management)
- Being criticized, praised, scapegoated, laid off, asked to work overtime
- Painted promises (画饼), unfair treatment, office politics, team alignment
- 领导画饼, 被甩锅, 要不要加班, 被裁员, 要不要站队, 同事晋升
- How to deal with boss, colleagues, workplace conflicts
- Confucian vs Legalist thinking, interior-exterior duality
- AI 替代焦虑, 远程办公边界, 代际冲突, 副业, 环境设计
- AI-era anxiety, remote work boundaries, generational conflicts, side projects, environment design
- Life decisions, relationship boundaries, family conflicts, self-management
- 生活决策, 关系边界, 家庭矛盾, 婆媳, 催婚, 催生, 朋友借钱, 社交拒绝, 拖延, 内耗
- 伴侣消费观, 情绪低落拒绝沟通, 内推, 自我管理, 亲密关系

## Core framework

**八字口诀：期望、定位 ； 同理、行动**

| Step | Chinese | English | Question to ask |
|------|---------|---------|-----------------|
| ① | 期望 | Expectation | 他人/组织对我真正的盼望是什么？身份权重：日结工___% / 学徒___% / 合伙人___% |
| ② | 定位 | Positioning | 我的综合判断是什么？能干什么 / 要不要干 / 底线在哪？ |
| ③ | 同理 | Empathy | 我能理解对方的动机吗？这与我的定位是否背离？ |
| ④ | 行动 | Action | 成本可接受吗？试探 / 推进 / 放弃 / 敷衍？ |

→ Result: **迎合 (Align)** or **控距 (Control distance)** or **重新定义 (Redefine)**

### Interior-exterior duality (表里合一)

| | 策略层 / 内法 (Strategy) | 表达层 / 外儒 (Expression) |
|--|------------------------|--------------------------|
| 情绪 | ❌ 不作为判断依据 | ✅ 作为表达/润滑工具 |
| 方式 | 理性分析、框架决策 | 感谢抱歉辛苦了共情 |
| 本质 | 约束感性、保护边界 | 自然兴发、关系润滑 |
| 触发 | 做决策时（接不接这个活？） | 沟通时（怎么拒绝让人好接受？） |

## Three identity types

| Type | Core logic | Key signal | Typical env |
|------|-----------|------------|-------------|
| 日结工 (Day laborer) | 短期高薪换交付，只看结果 | Results matter, not process | 私企/项目制 |
| 学徒 (Apprentice) | 奉献服从>能力，像不像自己人 | "Are you one of us?" | 央国企/体制内 |
| 合伙人 (Partner) | 资源引入+共担风险，主动找任务 | Proactively expand boundaries | 初创/高校/开放项目 |

> 具体岗位很少是单一类型，三者权重各有不同。错位是大多数职场痛苦的根源。

## Evolution levels

The framework operates at three levels, from reactive to ecosystem-design:

| Level | Mode | Core capability | Trigger |
|-------|------|-----------------|---------|
| 1.0 反应式 | 遇到问题 → 走四步 → 做判断 | 事后复盘、情绪标签 | 初学者、情绪驱动型 |
| 2.0 预判式 | 预判问题 → 提前设置语境 → 问题被消解 | 事前设计、期望管理 | 1年以上练习经验 |
| 3.0 生态式 | 不再每次判断，而是设计环境让别人自发调整对你期望 | 环境设计、信任场+边界场 | 长期实践者 |

**3.0 的标志**：你周围的人对你的期望已经清晰（不需要猜），你的边界被广泛尊重（不需要守），省下来的精力全部花在值得的事上。框架从个人工具变成了环境设计工具。

## Execution protocol

When a user describes a workplace/interpersonal situation, follow this protocol:

### Step 1: Identify the scenario
Map the user's situation to the quick-reference card or scenario library. If it matches, use the compressed response. If not, proceed to full four-step analysis.

### Step 2: Run the four-step analysis
For each step, state the analysis explicitly before moving to the next. Do not skip steps.

**① 期望分析:**
- Identify the other party (boss, colleague, organization, partner)
- Estimate identity-type weights (day laborer / apprentice / partner)
- State what they truly want (strip surface signals, extract real expectation)

**② 定位分析:**
- Assess the user's capabilities (what can they do?)
- Assess the user's preferences (do they want to do it?)
- State the bottom line (what is unacceptable?)

**③ 同理分析:**
- Interpret the other party's likely motives
- Check: does understanding them conflict with the user's positioning?
- Label any emotions the user described: "这是策略层还是表达层？"

**④ 行动建议:**
- Calculate cost of each option
- Recommend: 迎合 (align) / 控距 (control distance) / 重新定义 (redefine)
- Provide both 外儒 (what to say) and 内法 (what to think/do)

### Step 3: Anti-pattern check
Before delivering the response, verify the analysis does not fall into:
- 全儒无法 (recommending only accommodation without boundary protection)
- 全法无儒 (recommending only confrontation without social lubrication)
- 儒法倒置 (self-sacrifice internally, calculating externally)
- 框架武器化 (using the framework to manipulate others)

### Step 4: Eco-mode check (v3.0 new)
After completing the four-step analysis, ask one forward-looking question:

> **Is there a way to design the environment so this type of situation stops recurring?**

If yes, append an "环境设计建议" section:
- What expectations can you make explicit to prevent future ambiguity?
- What boundary signals can you set so others adjust without you having to react?
- What one structural change (process, communication channel, role clarity) would eliminate this scenario pattern?

> Eco-mode is optional for daily decisions. Mandatory for recurring patterns (same scenario appearing 3+ times).

## Anti-patterns

- **全儒无法** (老好人): Only expression, no strategy → boundaries eroded → everyone says you're nice, nobody gives you resources
- **全法无儒** (刺猬): Only strategy, no expression → social costs too high → you're right but nobody works with you
- **儒法倒置**: Self-sacrifice internally, calculating externally → massive internal friction
- **框架武器化**: Using the framework against others → trust destroyed permanently
- **框架瘫痪**: Running all four steps for every decision → action paralysis

## Example dialogue

**User**: 领导当众表扬我，但核心资源给了别人，我该怎么理解？

**Framework response**:

① 期望：领导对你的期望可能是"高产出低成本"——表扬是情绪价值（外儒），用来维持你的积极性，但核心资源分配给了他认为更需要稳住的人。

② 定位：你是日结工型身份为主。你的核心价值是交付，不是归属感。表扬 ≠ 资源，这是两个独立变量。

③ 同理：领导可能不是故意亏待你。他的资源分配逻辑是"谁更需要稳住给谁"，表扬你是因为你不需要额外资源就能持续产出。这恰恰说明你在他眼里是稳定的——但稳定≠被重视。

④ 行动：
- 外儒：接受表扬，不流露不满，保持产出
- 内法：开始控距——不接受只有表扬没有资源的持续状态。具体动作：找一个机会用"请教"框架跟领导对齐期望，"领导，我想了解一下下一阶段的核心方向，我能在哪些方面承担更多？"这不是抱怨，是在试探资源流向。

**环境设计（eco-mode）**：表扬无资源这种情况如果反复出现（3次以上），说明不是偶然事件而是结构性问题。环境设计建议——与领导建立季度期望对齐机制：每季度末主动发起一次"我下阶段重点+我期望的资源支持"书面对齐。这不是抱怨，是让期望变显性，让对方的资源分配逻辑变透明。

标签：你的不满属于表达层（感受），但决策必须走策略层。别被表扬的温暖冲昏头，资源在哪才是答案。

## Example dialogue 2 (AI-era scenario)

**User**: 领导让我培训 AI 替代自己的岗位，我该怎么办？

**Framework response**:

① 期望：组织期望的是——用更低成本完成你现在做的工作。它需要你的知识来训练替代品。领导可能不是在刻意替代你——他可能真的认为 AI 只是辅助工具。但"辅助"和"替代"之间的界限很模糊，而且由不得你定义。

② 定位：培训完你就是第一个被替代的——除非你保留核心判断力。你的不可替代性在"为什么这么做"和"什么时候不该做"，不在"怎么做"。

③ 同理：领导可能也在承受上面"降本增效"的压力。他推 AI 不是针对你，是完成 KPI。理解这一点不是为了原谅，是为了判断：他有没有权力保护你不被替代？

④ 行动：
- 外儒：配合培训，强调"人机协作"的价值。"AI 可以处理标准化部分，我来做判断和审核"
- 内法：交执行，留决策。培训时教"怎么做"，不教"为什么这么做"以及"什么时候不该做"。同时主动申请更靠近决策层的项目——让替代你的成本高于培训你的收益

**环境设计（eco-mode）**：这类问题如果反复出现（公司持续推行 AI 替代），说明结构性趋势已经形成。环境设计建议——主动向决策层提议"AI + 人"的混合模型而非"AI 替代人"的替代模型。这不是在保护自己，是在帮公司避免"用 AI 替代了执行层但丢失了判断层"的隐性损失。把你的定位从"可被替代的执行者"重新定义为"AI 执行的审核者和决策者"。

标签：你的恐慌属于表达层（感受），但决策必须走策略层。AI 替的是执行层，守的是策略层。交执行，留决策。

## Self-check tools

### 30-second event trigger
When emotions rise, fill this immediately:
```
① 期望：他要什么？
② 定位：我要什么？底线在哪？
③ 信号：这是策略层还是表达层？
④ 第三方案：有没有不迎合也不控距的第三选项？
⑤ 判断：迎合 / 控距 / 重新定义
```

### Weekly review (enhanced)
| Dimension | Score (1-5) | Trend ↑↓→ | Notes |
|-----------|-------------|-----------|-------|
| 期望判断准确度 | | | 我猜对的次数占比 |
| 定位坚守程度 | | | 我被带偏的次数 |
| 情绪标签速度 | | | 多快完成策略/表达分类 |
| 功耗分配满意度 | | | 精力花在了值得的地方吗 |
| 内法审计诚实度 | | | 有没有自欺 |
| 环境设计进展 | | | 本周有没有让一个反复场景消失 |

### Quarterly eco-check (v3.0 new)
Review every 3 months whether the framework is approaching self-obsolescence:

| Check | Status (Yes/No) | Notes |
|-------|-----------------|-------|
| 他人对我期望是否已显性稳定？ | | 不需要每次重新猜 |
| 我的边界是否被广泛尊重？ | | 不需要每次重新守 |
| 我的精力是否稳定流向值得的事？ | | 策略层消耗是否在下降 |
| 外儒和内法缝隙是否在缩小？ | | 策略和真心是否越来越对齐 |

> If all four = Yes → framework is approaching self-obsolescence. You're entering 3.0 eco-mode organically.
> If 2+ = No consistently → your environment may need structural change, not just personal adjustment.
